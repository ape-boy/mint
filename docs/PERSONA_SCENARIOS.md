## 2. 테크 리드 박팀장 (Tech Lead)

### 프로필
- **역할**: BM1743 제품군 테크 리드
- **담당 프로젝트**: 5개
- **주요 업무**: 빌드 품질 관리, 릴리즈 승인

### 2.1 팀 빌드 현황 확인

**오전 9시 - 팀 빌드 상태 점검**

```
시나리오: 팀의 5개 프로젝트 빌드 상태를 한눈에 확인하고 싶다.
```

**[현재 UX 시뮬레이션]**
```
1. "PROJECT" → "Project List"
2. P/J 필터에서 "BM1743" 선택
3. 테이블에서 5개 프로젝트 확인
4. "Result" 컬럼에서 각 프로젝트 빌드 상태 확인
   - BM1743_LENOVO_1TB: success
   - BM1743_DELL_2TB: failed
   - BM1743_HP_4TB: running
   - ...

소감: "필터로 내 팀만 보는 건 좋은데, 빌드 결과 컬럼이 작아서 보기 어렵다"
```

**[아쉬운 점]**
- 빌드 상세 (어떤 단계에서 실패했는지)를 보려면 각 프로젝트 진입 필요
- "내 팀" 필터를 매번 설정해야 함
- 실패한 프로젝트가 눈에 띄지 않음 (빨간색 강조 필요)

**[개선 제안]**
```
1. Dashboard → "My Team Projects" 섹션
2. 실패 빌드가 상단에 강조 표시
3. 실패 이유 (단계) 바로 표시
4. 클릭 시 해당 프로젝트 빌드 상세로 이동
```

---

### 2.2 릴리즈 승인 (TR)

**오후 4시 - 릴리즈 요청 처리**

```
시나리오: BM1743_LENOVO_1TB의 Release 빌드가 모든 조건을 통과했다.
         TR(Test Request)을 승인해야 한다.
```

**[현재 UX 시뮬레이션]**
```
1. Project List → BM1743_LENOVO_1TB
2. 빌드 탭 이동
3. Release Layer 선택
4. 최신 빌드 클릭
5. 우측 패널에서 "Release Status" 확인
6. "Release Criteria: PASSED" 확인
7. "TR (Test Request)" 버튼 클릭
8. 확인 모달 → 승인

총 클릭: 6-7회
소감: "TR 버튼을 찾기 쉽고, Criteria 확인도 한눈에 된다"
```

**[만족스러운 점]**
- Release Criteria 상태가 명확
- TR 버튼이 조건 충족 시에만 활성화

**[아쉬운 점]**
- 여러 프로젝트 TR 처리 시 각각 들어가야 함 (일괄 처리 불가)

- 팀/제품군 레벨 품질 대시보드 부재

## 5. 시나리오 기반 UX 개선 요약

### 5.1 공통 Pain Points

| 문제 | 영향받는 사용자 | 우선순위 |
|------|---------------|----------|
| 최근/즐겨찾기 프로젝트 접근 없음 | 개발자 | 높음 |
| 팀/제품군 대시보드 부재 | TL, PM | 높음 |
| 일괄 작업 불가 | PM, TL | 매우 높음 |
| 빌드 알림 없음 | 전체 | 중간 |
| 프로젝트 생성 불가 | Admin | 낮음 (운영 단계에서) |

### 5.2 페르소나별 핵심 개선 제안

| 페르소나 | 핵심 니즈 | 개선 방향 |
|---------|----------|----------|
| 개발자 | 빠른 빌드 실행 | "My Projects" 위젯, 즐겨찾기 |
| TL | 팀 현황 한눈에 | Task Group 대시보드, 실패 하이라이트 |
| PM | 전사 현황 + 일괄 작업 | Batch Center, 마일스톤 경고 |
| Admin | 프로젝트 관리 | (후순위) 관리 기능 |

### 5.3 Quick Win 목록

1. **Dashboard에 "My Recent Projects" 추가**
   - localStorage로 최근 접근 프로젝트 5개 저장
   - 빠른 접근 링크 제공

2. **Project List에 빌드 상태 하이라이트**
   - 실패 빌드 빨간색 강조
   - 실패 이유 툴팁

3. **필터 기억 기능**
   - 마지막 사용한 필터 저장
   - "내 필터" 저장 기능

4. **빌드 실패 시 브라우저 알림**
   - Web Notification API 활용
   - 백그라운드에서도 알림

## 6. 결론

### 핵심 인사이트

1. **개발자 일상 업무는 잘 지원됨**
   - Split View 빌드 관리가 효과적
   - Layer 전환 UX 양호

2. **TL/PM 업무 지원 부족**
   - 팀/전사 레벨 뷰 부재
   - 일괄 작업 불가가 심각한 병목

3. **특수 시나리오 대응 불가**
   - 공통 패치 일괄 릴리즈 시나리오
   - 긴급 상황 대응 어려움

### 권장 로드맵

```
Phase 1 (1-2주): Quick Wins
├─ My Recent Projects 위젯
├─ 필터 기억 기능
└─ 빌드 상태 하이라이트

Phase 2 (3-4주): Core Features
├─ Task Group 대시보드
├─ Batch Build 기능
└─ 웹 알림 기능

Phase 3 (5-8주): Advanced
├─ Batch TR 기능
├─ 커스텀 대시보드
└─ 관리 기능 (프로젝트 CRUD)
```

이 문서는 실제 사용자 인터뷰로 검증이 필요합니다.
